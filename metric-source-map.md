# Stock Dashboard Metric Source Map

작성일: 2026-04-29
목적: 무료 우선으로 1분 갱신 주식/매크로 대시보드의 지표별 데이터 소스 후보, 티커 후보, 실현 가능성, 리스크를 정리한다.

## 설계 원칙

1. **무료 우선**: 유료 API 없이 MVP를 만든다.
2. **1분 갱신 목표**: 단, 장이 열린 지표만 1분 갱신한다.
3. **장마감/휴장 감지**: 장이 닫힌 지표는 회색 `장종료` 상태로 두고 불필요한 호출을 줄인다.
4. **데이터 상태 표시 필수**:
   - 🟢 정상/개장/갱신중
   - 🟡 지연/대체소스/검증필요
   - ⚪ 장종료/휴장
   - 🔴 수집실패/티커불명
5. **AI 비사용 원칙**: 1분 데이터 갱신에는 AI를 호출하지 않는다. AI는 이상징후 요약/주간 브리핑/사용자 질의 때만 사용한다.
6. **스크래핑은 최후순위**: 공식/준공식 API와 yfinance로 안 되는 지표만 제한적으로 검토한다.
7. **네이버증권 모바일은 후보 소스로 추가**: 다니가 전체 지표가 네이버증권 모바일 검색에서 실시간 표시됨을 확인했다. 단, 자동 수집은 약관/차단/구조변경 리스크가 있어 보조·검증·fallback 후보로 둔다.

## 무료 운영 리스크

1. yfinance는 공식 Yahoo 제휴 API가 아니며 개인/연구용 성격이 강하다.
2. 무료 소스는 지연·누락·티커 변경·차단 가능성이 있다.
3. 장시간을 고려하지 않고 모든 지표를 24시간 1분 호출하면 제한에 걸릴 가능성이 크다.
4. 채권/한국 파생/국내금은 무료 1분 실시간 구현의 핵심 난점이다.
5. 스크래핑은 가능하지만 장기 안정성은 낮다. 지표별 상태 표시와 fallback이 필수다.

## 다음 검증 작업

1. KIS Open API에서 코스피200선물/야간선물/채권 시세 가능 여부 확인.
2. 한국채·일본채 무료 티커 후보 추가 검색.
3. 국내금의 기준을 정하기: KRX 금시장 vs 국제금 원화환산.
4. 네이버증권 모바일에서 전체 지표가 실시간 조회되는 점을 바탕으로, 자동 수집 가능성/약관/차단 리스크를 별도 검토.
5. 각 지표별 거래시간 캘린더 생성.
6. `fetch_market.py` 프로토타입으로 yfinance/Binance 실제 1분 수집 테스트.
